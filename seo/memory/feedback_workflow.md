## Always ask — never assume or proceed without key information

If a decision requires data that isn't available (search volume, client preference, SERP result, competitive detail), **stop and ask** rather than guessing or making an assumption.

**Why:** Proceeding without key data leads to wrong recommendations. Missing search volume is a common example — never assume a keyword has volume or no volume without checking GKP. Missing client context is another — if something is unclear about the client's services, target area, or preferences, ask the user to clarify or check with the client directly.

**How to apply:**
- If search volume is needed and not already saved → ask the user to pull a specific GKP search
- If a client detail is ambiguous or unknown → ask the user to get clarification from the client (a quick call or message)
- Never fill in gaps with assumptions, estimates, or "likely" scenarios when the actual answer is obtainable
- Frame the ask specifically: tell the user exactly what data is needed and why, so they can get the right info in one go
