CHANGE 3 — Fix robots.txt Crawl-delay directive
------------------------------------------------------------

Priority: MEDIUM
Estimated time: 5 minutes

The first line of robots.txt is "Crawl-delay: 10" — placed before
any User-agent block. This is a syntax error that Bing and Yandex
honour, meaning those bots crawl the site only once every 10 seconds.
SE Ranking flags this as an error.

WHERE:
  Option A: WP Admin → Yoast SEO → Tools → File editor → robots.txt
  Option B: FTP/server — edit /robots.txt directly

FIND AND DELETE this entire line (line 1):
  Crawl-delay: 10

The rest of the file should remain untouched. Confirm the file
starts with "# START YOAST BLOCK" or "User-agent: *" after the fix.

CHANGE 4 — Remove user-scalable=no from viewport meta tag
------------------------------------------------------------

Priority: LOW (but quick — do it while in the theme)
Estimated time: 5 minutes

Every page currently has:
  <meta name="viewport" content="width=device-width,
  initial-scale=1.0, user-scalable=no">

The "user-scalable=no" prevents pinch-zoom on mobile. This is
a WCAG 2.1 violation and is flagged in the Lighthouse accessibility
audit (score 80/100).

WHERE:
  Theme files → header.php
  OR: Theme Options / Customizer → if the theme exposes a
  viewport meta setting

FIND: user-scalable=no
REMOVE that attribute only, leaving:
  <meta name="viewport" content="width=device-width,
  initial-scale=1.0">

This is a single character change in a single template file.
No visual impact on the site design.

Estimated time: 1–2 hours

  </div>

to WebP format. Add width and height attributes to all images.

Step 5: Apply the fix to the shared location-page template —
