Technical SEO

3 Critical Issues
  • Four key service URLs return 404 errors
    /skylight-installation/, /skylights/, /skylight-replacement/, and /about/ all return 404. Crawlers and users hitting these URLs get dead ends. /skylight-replacement/ in particular is a business-critical service URL.
    Action: Create /skylight-replacement/ page or 301 redirect these to correct destinations
  • robots.txt contains suspicious staging/test Disallow paths
    14 Disallow entries including /adpromedia, /andy, /cssite, /viet-times, /hobartskylights, /saigonxamthammy, /skymarble, /skylightsadelaide — these suggest old staging directories or test content still on the server. Security and crawl budget risk.
    Action: Audit what exists at these paths; remove/clean up any remaining test content
  • Sitemap includes non-indexable utility pages
    page-sitemap.xml includes /checkout/, /cart/, /my-account/, /cancel-payment/, /my-bookings/, /bookingpress-complete-payment-2/, /appointment-cancellation-confirmation/ (listed twice). These pages should be excluded from sitemaps or given noindex tags.
    Action: Exclude WooCommerce utility pages from sitemap; add noindex to transactional/system pages
  • 11 broken internal pages + 24 broken backlinks detected
    DataForSEO crawl identified 11 internally broken pages and 24 external backlinks pointing to deleted/changed pages. Broken internal links waste crawl budget; broken backlinks lose link equity.
    Action: Run full crawl to identify broken internal links; create 301 redirects for changed URLs with backlinks
  • Backlink profile dominated by one low-quality domain (vtimes.com.au)
    vtimes.com.au has sent 45,319 links to centralskylights.com.au — 80% of all backlinks from a single domain. This appears to be a link farm/directory network. Despite 279 referring domains, domain rank is only 394 — lower than competitors with far fewer links. This link imbalance may be holding back domain authority.
    Action: Investigate vtimes.com.au relationship; consider disavow if it is a purchased link network
  • Page speed data unavailable (Lighthouse/PSI API requires key)
    Lighthouse could not be run in this environment. Recommend running a manual PageSpeed Insights audit at pagespeed.web.dev for both mobile and desktop scores. Given 188 internal links on homepage and 19 images, check for render-blocking issues.
    Action: Run manual PSI check; target LCP < 2.5s, CLS < 0.1, TBT < 200ms
  • HTTPS and redirect chain working correctly
    HTTP → HTTPS 301 redirect functions correctly. All main service pages return 200. Rank Math SEO plugin in use — good foundation.

Local SEO

NAP Conflict — Fix Immediately
Address Conflict: The LocalBusiness schema on the homepage says the address is "88 Cheltenham Road, Dandenong VIC 3175". The contact page and Google Business Profile both show "2/21 Tower Court, Noble Park VIC 3174". These are different streets, different suburbs, and different postcodes. Google sees conflicting signals which directly suppresses local pack eligibility.

Google Business Profile

  • Status: Active (Knowledge Graph confirmed)
  • Listed as: Skylight contractor in Noble Park
  • Rating: 5.0 stars (313 reviews)
  • CID: 15060042115388287151
  • Not appearing in local pack for "skylight installation melbourne"

NAP Sources

  • Contact page: 2/21 Tower Court, Noble Park 3174
  • Schema (homepage): 88 Cheltenham Rd, Dandenong 3175
  • GBP: Noble Park (matches contact page)
  • Schema reviews: 293 (GBP shows 313)
  • Schema must be updated to Noble Park 3174

Schema Issues to Fix

  • LocalBusiness schema address wrong
    streetAddress: "88 Cheltenham Road" → should be "2/21 Tower Court"
    addressLocality: "Dandenong" → should be "Noble Park"
    postalCode: "3175" → should be "3174"
  • openingHoursSpecification format error
    "opens": "08.30" must be "08:30" (colon not period) for valid parsing
  • aggregateRating count out of date
    Schema shows 293 reviews but GBP has 313. Update to current count monthly.
  • No LocalBusiness schema on /contact/ page
    This is the most important page for local signals — add full LocalBusiness schema here.
  • Schema @type is generic "LocalBusiness"
    Should use more specific type: "HomeAndConstructionBusiness" or "GeneralContractor" to better signal service category.

Review Strength — Key Asset

313 x 5-star Google reviews is exceptional for a Melbourne skylight contractor. This is a significant competitive advantage. The reviews are also surfacing as a Google Reviews widget in SERP (confirmed). Leverage this in all meta descriptions and landing page copy.
