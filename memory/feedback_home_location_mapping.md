---
name: Home location maps to homepage, not location page
description: For local businesses, the suburb where the business is physically located should be mapped onto the homepage or main service page, not a standalone location page
type: feedback
---

For local businesses, the business's home suburb/address should be incorporated into the homepage entity (NAP, schema, content, keyword signals) — not created as a standalone location page.

**Why:** A standalone location page for the home suburb is redundant — the homepage already signals that location through address, schema, and NAP consistency. A separate page competes with the homepage and wastes a location budget slot on a page that will never outrank the homepage for the same location.

**How to apply:** When building location page sets, check if any suburb in the list matches the business's physical address. If yes, remove it from the location page roster, incorporate the location into the homepage strategy (as a secondary keyword signal or content entity), and reallocate those freed budget slots to a new suburb or service page.
